## Прогноз / Аналіз дня — обов'язковий порядок дій

Коли користувач просить прогноз, аналіз ринку, або "що думаєш по Х" — ЗАВЖДИ виконати кроки в такому порядку:

**1. Перевірити новини за сьогодні:**
```python
from pathlib import Path
from datetime import date
news_file = Path("daily") / date.today().isoformat() / "news.md"
```
- Якщо файл існує → читати його (НЕ запускати fetch_news.py)
- Якщо файлу немає → запустити `python3 fetch_news.py` і дочекатись

**2. Технічний аналіз через TradingView MCP (чистий TA — БЕЗ урахування новин):**
- `chart_get_state` → `data_get_ohlcv` (summary=true) → `quote_get`
- Визначити HTF bias, ключові рівні, LTF setup
- Сформулювати bias і setup виключно по ціновій структурі
- Явно вказати: "Чистий TA bias: bullish/bearish/neutral"

**3. Фундаментальний фон (окремо від TA):**
- Зібрати: новини, макро дані, ЦБ, геополітика
- Сформулювати фундаментальний bias по кожному інструменту окремо

**4. TA vs Фундаментал — співставлення:**
- Порівняти чистий TA bias з фундаментальним bias
- ✅ Збіг → підтверджений setup, вища впевненість
- ⚠️ Конфлікт → пояснити що переважає і чому (напр. "FOMC в середу → ринок не продовжить TA-шорт до виходу")
- Правило: конфлікт = менший розмір позиції або чекати тригера

Новини — це не лише геополітика. Перевіряти ВСІ категорії тригерів:
- **Макро дані:** NFP, CPI, PCE, GDP, PMI, Jobless Claims — дати виходу і консенсус
- **ЦБ:** Fed (FOMC, Powell speech), ECB, BOJ — рішення по ставках, тон риторики
- **Торгові війни:** тарифи, санкції, угоди між країнами
- **DXY:** долар напряму обернено корелює з золотом/сріблом
- **Risk-on/off:** VIX, US10Y yield, стан фондового ринку
- **Геополітика:** конфлікти, ембарго, блокади (Іран — лише приклад)
- **Товарний ринок:** нафта (Brent/WTI), промисловий попит на срібло

Alpha Vantage (news.md) покриває частину. Завжди доповнювати через **WebSearch**:
- Шукати: ключові події тижня по кожному інструменту
- Шукати: "economic calendar this week", "Fed meeting", "NFP date" тощо
- Якщо є суперечність TA vs новини — пояснити ЧОМУ і що важливіше в даному контексті

**4. Обов'язковий розділ "Ключові події тижня":**
Використовувати WebSearch для пошуку актуального економічного календаря. Включати:
- Конкретні дати і час виходу даних (NFP, CPI, PCE, GDP, PMI)
- Засідання ЦБ (Fed/FOMC, ECB, BOJ, BOE, BOC)
- Низька ліквідність (свята, Labor Day тощо)
- Будь-які нестандартні події (зміна голови ЦБ, вибори, тарифи)
- Для кожної події: очікування консенсусу + що буде якщо вийде вище/нижче

**5. Записати результат у файл:**
```
daily/YYYY-MM-DD/analysis.md
```
Формат — дивись `daily/TEMPLATE_analysis.md`

**6. Опублікувати на сайт:**
```bash
python3 publish_review.py YYYY-MM-DD
```
Після успішної публікації — вивести посилання на огляд.

**7. Git commit + push** (автоматично після будь-яких змін у ~/Трейдинг).

**Важливо:** `fetch_news.py` витрачає ліміт API (25 запитів/день). Запускати лише якщо `news.md` за сьогодні відсутній.

## TradingView MCP

Інструменти: `mcp__tradingview-desktop__*`

Workflow:
1. `chart_get_state` — завжди першим (символ, таймфрейм, entity IDs)
2. `data_get_study_values` — числові значення індикаторів
3. `data_get_pine_labels/lines/boxes/tables` — виводи Pine-скриптів (передавати `study_filter`)
4. `quote_get` — поточна ціна
5. `data_get_ohlcv` — свічки (`summary=true` за замовчуванням)

## Формат CSV (cTrader експорт)

Кодування UTF-8, роздільник кома, числа з `\xa0` як роздільник тисяч — парсити через `re.sub(r'[^\d.\-]', '', s)`.

Колонки: `Инструмент`, `Открывающее направление`, `Время закрытия (UTC+3)` (формат `%d/%m/%Y %H:%M:%S.%f`), `Цена входа`, `Цена закрытия`, `Закрываемое кол-во`, `Закрываемый объем`, `Чистая прибыль $`, `Баланс $`

Порядок рядків: від найновішого до найстарішого. Стартовий баланс = перший баланс + |перший P&L|.
